# Веб-симулятор: Свёрточный код + декодер Витерби

Это статический сайт (HTML + CSS + JavaScript) без серверной части.  
Он моделирует цепочку:

**Свёрточный кодер (rate 1/2, k=1, n=2) → канал с ошибками (BSC) → декодер Витерби (hard-decision)**

## Возможности
- Ввод бинарной строки (только 0 и 1)
- Генераторы в **восьмеричной** форме (например `7,5`) или **бинарной** (`111` и `101`)
- Ошибки в канале:
  - случайно по BER (0…0.3)
  - фиксированное количество ошибок
  - кнопка «Сгенерировать новый шум»
  - seed (необязательно): одинаковый seed → одинаковые ошибки
- Вывод результатов:
  - Исходная
  - Закодированная (пары бит)
  - Искажённая
  - Восстановленная (Витерби)
  - процент ошибок канала и восстановления
  - подсветка отличающихся бит
- Таблица шагов (упрощённый лог Витерби)

## Как запустить локально (самый простой способ)
### Вариант 1: просто открыть файл
1. Открой `index.html` двойным кликом.
2. Всё должно работать.

### Вариант 2 (рекомендуется): через Live Server в VS Code
1. Установи расширение **Live Server** (Ritwick Dey).
2. Открой папку проекта в VS Code.
3. Открой `index.html` → нажми **Go Live** (внизу справа).

## Публикация на GitHub Pages
1. Создай репозиторий на GitHub (например `viterbi-simulator`).
2. Загрузите файлы `index.html`, `styles.css`, `app.js`, `README.md` в корень репозитория.
3. Открой репозиторий → **Settings** → **Pages**
4. **Build and deployment**:
   - Source: **Deploy from a branch**
   - Branch: `main` / `(root)`
5. Подожди, пока появится ссылка вида:
   `https://USERNAME.github.io/REPO/`

Если видишь 404 — проверь:
- что Pages включён именно для ветки `main`
- что файлы лежат в корне репозитория
- что главный файл называется именно `index.html`
